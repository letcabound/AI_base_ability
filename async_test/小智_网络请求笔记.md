# TCP 和 UDP
TCP协议 通过三次握手和四次挥手建立和关闭TCP连接。数据丢失会重发直到对方确认或超时断开。常用于 网站浏览，文件传输，数据库连接等。
UDP协议 不建立也不维护连接，将数据发送到制定目标IP和端口就不管了，但也会受到网络可达性和权限的约束。数据丢失不重传，但是低延迟低开销，用于直播/游戏等。

# HTTP、HTTPS 和 websocket 和 流式传输 
HTTP和websocket最主要的区别就是：websocket是双工通道，客户端和服务端都可以主动向对方发送信息。
而HTTP只有在客户端主动向服务端发起请求后，服务器才可以被动响应向客户端发送信息，服务器不能主动向客户端发送信息，除非 SSE 。

Websocket 和 HTTP，底层都是通过TCP协议实现的，都能够实现流式传输。而Websocket双工通道，更适合用来 在线聊天，网络游戏等。

Websocket连接建立过程：客户端先通过 HTTP 发起一个升级请求（Upgrade: websocket）。
服务器同意后，连接复用原有的 TCP 连接（已经完成三次握手），协议升级为 WebSocket。

HTTPS和HTTP相比，建立连接时在三次握手的基础上加上TLS握手，然后发送HTTP请求。
